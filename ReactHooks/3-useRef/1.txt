[ReactHooks]

- 사용법
1. const ref = useRef(value) → { current: value }
	$ 인자로 넣어준 초기값 value는 current에 저장이 된다.
	$ ref 오브젝트는 언제든 수정이 가능하며, 원하는 값으로 바꿀수 있다.
	$ 반환된 ref는 컴포넌트의 전 생애주기를 통해 유지가 된다.
	$ 즉, 컴포넌트가 계속 렌더링이 되어도 컴포넌트가 언마운트 되기 전까지는 값을 계속 유지할수 있다.

- 언제 사용 되느냐? useRef가 유용한 2가지 상황1
	$ 1. Ref는 state와 비슷하게 어떠한 값을 저장하는 저장공간의 역할을 한다.

	# state의 경우
	$ State가 변화 ► 렌더링 ► 컴포넌트 내부 변수들 초기화
	* 컴포넌트의 내부 변수가 초기화 되면 원하지 않는 렌더링 때문에 곤란해진다.

	# state대신 Ref에 값을 저장하는 경우의 장점
	$ Ref가 변화 ► No 렌더링 ► 변수들의 값이 계속 유지됨(불필요한 렌더링을 막는다.)
	$ 그리고 state가 변화, 바뀌어서 렌더링 된다해도 Ref의 값은 계속 유지되는 장점이있다.
	$ 그렇기 때문에 변경시 렌더링을 발생시키지 말아야 하는 값을 다룰때 편리하다.
